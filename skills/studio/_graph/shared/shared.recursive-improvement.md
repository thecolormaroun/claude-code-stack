# Recursive Improvement Loop

Pattern:

`generate → evaluate → diagnose → improve → repeat (until passing)`

## Use when
- the output will be read by humans (customers, team)
- persuasion/clarity matters
- quality is visible and compounding

## How to run it
1) **Generate v1**
2) **Evaluate**: score each criterion 1–10 (be brutal)
3) **Diagnose**: write why low scores are low + what "passing" looks like
4) **Improve**: rewrite (don't patch)
5) **Repeat** until all criteria ≥ 8/10

## Optional: adversarial pass
- Skeptical reader
- Distracted scroller
- Competitor teardown

Store the criteria as reusable checklists per asset type.
