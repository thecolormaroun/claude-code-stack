# Persona Voice Guide

Every product has a personality, even if it was never defined. Define it intentionally — or the copy will default to Generic Corporate.

---

## How to Use This File

1. Start with the **Voice Template** section below
2. Fill it out for each project persona
3. Reference the completed guide when writing copy for that project
4. When tone questions arise, ask: "What would [Persona Name] say here?"

---

## Example: Recipe App Voice Guide

*(Worked example for your active project — family meal planning app)*

### Brand Personality
Warm, practical, a little playful, parent-smart, never preachy.

### Voice Statement
"Recipe App sounds like your friend who's a great cook and a parent — they know feeding kids is hard, they don't judge, and they always have a good idea ready."

### This brand IS / IS NOT

| IS | IS NOT |
|----|--------|
| Warm | Clinical |
| Parent-smart | Preachy |
| Practical | Fussy |
| Lightly playful | Silly or juvenile |
| Encouraging | Judgy about food choices |
| Concise | Wordy |

### Tone Dial Settings
- Formality: **2/5** → Conversational, contractions everywhere
- Warmth: **5/5** → Always warm; this is about family, not productivity
- Playfulness: **3/5** → Light touches, never silly
- Directness: **4/5** → Parents don't have time; get to the point
- Technical depth: **1/5** → No cooking jargon beyond what parents already know

### Vocabulary

| Say this | Not this |
|----------|----------|
| Meal | Recipe entry |
| Save | Add to library |
| Noms | N/A (brand-specific, use sparingly) |
| Family | Users / Household |
| Add to this week | Schedule |
| Your kids | Children |
| Try this | Explore |

### The Test
When in doubt: "Would a warm, non-judgmental parent friend say this?"

### Copy Examples by Context

**Error:**
- ✅ "Couldn't save that recipe. Check your connection and try again."
- ❌ "Error: Recipe save operation failed."

**Success:**
- ✅ "Recipe saved! Find it in Your Collection."
- ✅ "Week planned. Enjoy the cooking."
- ❌ "Recipe has been successfully added to your collection."

**Empty State (no meals planned):**
- ✅ "Nothing planned this week yet. Add a meal to get started — or let us suggest something."
- ❌ "No meals scheduled."

**CTA:**
- ✅ "Add to this week"
- ✅ "Find a recipe"
- ✅ "Suggest something"
- ❌ "Submit" / "Confirm" / "Proceed"

**Onboarding:**
- ✅ "What do your kids actually eat? Let's start there."
- ✅ "Add your first recipe. It can be as simple as 'Tuesday pasta.'"
- ❌ "Complete your profile to begin using the application."

## Multi-Persona Projects

For products with distinct user types (e.g., admin vs. end user, parent vs. child):

```markdown
## [Role 1] Voice
[Fill template above]

## [Role 2] Voice
[Fill template above]

## Switching Modes
When writing copy, tag each string with the persona it addresses.
Never let Role 2 voice bleed into Role 1 surfaces.
```

---

## Registering a New Project Persona

When you starts a new project:
1. Duplicate the template section
2. Name it with the project name
3. Fill in personality, dials, vocabulary, and examples
4. All copy for that project runs through this guide

Projects without a completed voice guide default to: **Warm, Direct, Practical** — no personality, no problems.
