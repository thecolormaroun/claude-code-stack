# Design Inspiration Sources

These are the galleries, collections, and channels Studio should study for design intelligence. Not just "pretty sites" — these are sources that explain WHY design decisions work.

## Tier 1 — Primary Sources (check weekly)

### Full Site Design
- **Brian Lovin /sites** — https://brianlovin.com/sites
  - Curated by former GitHub/Meta designer. High bar. Real shipping products.
  - Best for: Overall site quality, navigation, layout, polish

### Micro-Interactions & Delight
- **Design Spells** — https://www.designspells.com/
  - "Design details that feel like magic" by Chester & Duncan
  - Best for: Motion, transitions, hover states, subtle polish, "wow" moments
  - This is what separates good from great

### Mobile Design
- **Handheld Design** — https://www.handheld.design/
  - Mobile design decisions you can explain to your PM. Weekly frameworks.
  - Best for: Mobile patterns, gesture design, decision rationale
  - Unique value: Explains the WHY, not just the WHAT

- **Featured Mobile** — https://featuredmobile.com/
  - Mobile app showcase
  - Best for: Mobile UI patterns, app design trends

## How to Use These

### During `/tastemining` or `/research`
1. Start with Tier 1 sources — highest signal
2. Scrape recent additions with Firecrawl
3. Extract patterns that match the project's direction
4. Save findings to `memory/reference/research/taste-[project].md`

### During weekly design digest (cron)
1. Check Tier 1 sources for new content
2. Extract 3-5 standout patterns or examples
3. File into CDO references by category
4. Note trends across sources

### When adding new sources
1. Add to this file in the appropriate tier
2. Do an initial deep scrape
3. Extract foundational patterns
4. Update relevant design skills if patterns are reusable
