# Compound Engineering Checklist

## Nightly Review Checklist

| # | Check | Pass/Fail |
|---|-------|-----------|
| 1 | **All sessions scanned** — Didn't miss any from last 24h | |
| 2 | **Patterns extracted** — At least noted "nothing new" | |
| 3 | **Errors logged** — Any failures added to ERRORS.md | |
| 4 | **Corrections captured** — User corrections → LEARNINGS.md | |
| 5 | **Daily note updated** — memory/daily/YYYY-MM-DD.md exists | |
| 6 | **Changes committed** — Git shows today's compound commit | |

## Extraction Quality Checklist

| # | Check | Pass/Fail |
|---|-------|-----------|
| 1 | **Specific not vague** — "Use haiku for crons" not "use right model" | |
| 2 | **Actionable** — Clear what to do differently | |
| 3 | **Context included** — When/where this applies | |
| 4 | **No duplicates** — Didn't re-add existing knowledge | |
| 5 | **Right destination** — Goes to correct file | |

## Promotion Checklist

Before promoting from .learnings/ to main files:

| # | Check | Pass/Fail |
|---|-------|-----------|
| 1 | **Pattern confirmed** — Seen 2+ times | |
| 2 | **No contradictions** — User hasn't objected | |
| 3 | **Still relevant** — Applies going forward | |
| 4 | **Right level** — AGENTS.md vs TOOLS.md vs skill file | |

## Weekly Review Checklist

| # | Check | Pass/Fail |
|---|-------|-----------|
| 1 | **Daily notes synthesized** — Week's highlights extracted | |
| 2 | **Multi-day patterns found** — Cross-session insights | |
| 3 | **MEMORY.md updated** — If anything significant | |
| 4 | **Stale content archived** — Old details moved out | |
| 5 | **Promotions done** — Learnings → main files if ready | |

## Quality Gates

### Gate 1: No Noise
```
Is this worth remembering in 30 days?
```
- [ ] Not obvious or generic
- [ ] Specific to user/context
- [ ] Actionable

### Gate 2: Right Place
```
Can I find this when I need it?
```
- [ ] Correct file selected
- [ ] Searchable terms used
- [ ] Not buried in noise

### Gate 3: Won't Conflict
```
Does this contradict anything existing?
```
- [ ] Checked existing knowledge
- [ ] Updated or replaced if needed
- [ ] No duplicate entries
